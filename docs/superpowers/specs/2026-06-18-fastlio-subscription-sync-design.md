# FAST-LIO 订阅双向同步设计

日期：2026-06-18

## 目标

在 ROS 页面和定位精度页面各显示一个相同的“FAST-LIO 里程计”复选框与“应用订阅”按钮。两页复选框实时双向同步，但只有点击任一页面的“应用订阅”才修改共享 `RosBridgeWorker` 的实际 FAST-LIO 订阅。

## 行为

- 两页复选框显示同一个待应用状态。
- 任一页勾选或取消时，另一页立即同步显示，但不立即变更运行时订阅。
- 点击任一页“应用订阅”后：
  - 勾选：订阅当前 FAST-LIO topic，并向定位页发送定位样本。
  - 取消：取消当前 FAST-LIO topic 订阅，定位页停止接收新样本。
- 修改 FAST-LIO topic 时：
  - 已应用启用：共享 worker 从旧 topic 切换到新 topic。
  - 已应用禁用：只保存 topic，不建立订阅。
- ROSbridge 连接和重启恢复最后一次已应用的 FAST-LIO 启用状态。
- ROS 页面普通数据 topic 列表不再建立第二条 FAST-LIO 订阅。

## 视频异常修复

定位页连接状态标签只由统一 ROSbridge 状态刷新路径写入。连接/健康事件不得再用“共享 ROSbridge 已连接”覆盖包含地址、延迟和时钟差的完整状态文本。

## 测试

- 定位页包含相同复选框和应用按钮。
- 两页复选框双向同步，切换本身不调用 worker。
- 任一页面应用后调用共享 worker 的 FAST-LIO 启用接口。
- Session 启用/禁用时建立或取消唯一 FAST-LIO 订阅。
- topic 在启用和禁用状态下切换行为正确。
- ROSbridge 重启恢复已应用状态。
- 定位页状态标签不再被健康刷新覆盖为简化文本。
